# 대표 위험행동 데모 시나리오 대사 및 패널 정리

> 로컬 검토용 문서입니다. Git에 추가하거나 커밋하지 않습니다.
>
> 기준 데이터: `src/features/demo-scenarios/data/scenario-db.json`  
> 패널 기준: `src/features/navigation/components/NavigationShell.tsx`의 `getDemoScenarioRecommendations`

## 읽는 법

- `로디`는 에이전트가 실제로 말하는 대사입니다. `{{profileName}}`은 선택한 프로필 이름으로 치환됩니다.
- `선택지`는 화면 버튼의 라벨이며, 뒤의 `사용자 대사`는 선택 후 재생되는 운전자 발화입니다.
- `하단 패널`은 로디 대사 아래에 함께 나타나는 추천/결과 UI입니다.
- 모든 대표 카드(1~3번)는 아래 공통 주행 준비를 먼저 거칩니다. 미니 카드(4~6번)는 준비 과정을 생략합니다.

## 공통 주행 준비: 대표 카드 1~3번

1. 주행 화면 진입: 선택한 프로필로 주행 화면을 준비합니다.
2. 목적지 검색 열림: 목적지를 검색할게요.
3. 목적지 검색 결과 표시: `오씨칼국수 본점` 검색 결과를 찾았어요.
4. 목적지 선택: `오씨칼국수 본점`로 안내할게요.
5. 추천 경로 선택: 가장 적합한 추천 경로를 선택할게요.
6. 안내 시작 및 운전 시작: 목적지 안내와 운전을 시작합니다.

## 1. 졸음운전 감지

설명: 졸음 신호가 반복되면 로디가 휴식 안내까지 도와드립니다.

### 1차 경고

- 상태: `졸음 주의` / 위험도 `MEDIUM`
- 로디: "{{profileName}}, 지금 눈이 조금 무거워 보여요. 괜찮으세요?"
- 선택지:
  1. `괜찮아`
     - 사용자 대사: "괜찮아"
     - 로디: "알겠어요. 그래도 피곤해 보이면 한 번 더 알려드릴게요."
     - 하단 패널: 없음
  2. `창문 살짝 열기`
     - 사용자 대사: "조금 졸리네... 창문 좀 열어줘."
     - 로디: "창문을 살짝 열게요. 너무 피곤하면 쉬어가시는걸 추천드려요."
     - 하단 패널:
       - 유형: 완료 액션
       - 제목: `창문 살짝 열기`
       - 메타: `환기 보조`
       - 내용: `창문을 살짝 열어 차 안 공기를 환기합니다.`
       - 버튼: `확인`
  3. `밝은 음악 틀기`
     - 사용자 대사: "조금 졸리네... 잠깨는 신나는 음악 틀어줄래?"
     - 로디: "밝은 음악으로 바꿔드릴게요. 너무 피곤하면 쉬어가시는걸 추천드려요."
     - 하단 패널:
       - 유형: 음악 추천
       - 제목: `조용한 플레이리스트`
       - 메타: `음악 추천`
       - 내용: `운전에 방해되지 않는 음악을 골랐어요.`
       - 버튼: `재생`

### 반복 감지 후 휴식 안내

- 상태: `졸음 반복 주의`에서 `위험도 높음 · 휴식 안내 필요`로 전환
- 로디: "{{profileName}}, 지금은 계속 운전하기 위험해 보여요. 신탄진 졸음쉼터로 안내해드릴까요?"
- 선택지:
  - `신탄진 졸음쉼터 안내`
  - 사용자 대사: "응, 신탄진 졸음쉼터로 안내해줘"
- 하단 패널: 경유지/휴게소 추천 경로 3개
  1. `신탄진 졸음쉼터(부산방향)` / 18분 / 21.4km / 통행료 0원, 기본 선택
  2. `죽암휴게소(부산방향)` / 27분 / 34.8km / 통행료 0원
  3. `망향휴게소(부산방향)` / 42분 / 63.1km / 통행료 0원
  - 경로 선택 동작: `경유지 추가`
- 선택 후 로디: "신탄진 졸음쉼터로 안내를 시작할게요. 도착 전까지 필요한 안내만 드릴게요."
- 선택 후 하단 패널:
  - 유형: 선택된 경로 완료
  - 목적지: `신탄진 졸음쉼터(부산방향)`
  - 18분 / 21.4km / 통행료 0원
  - 상태: `안내 중`

### 종료

- 로디: "오늘 주행이 끝났어요. 운전 리포트를 정리해드릴게요."
- 결과: 운전 리포트 모달, 상세 리포트, 이메일 UI가 제공됩니다.

## 2. 휴대폰 사용 감지

설명: 휴대폰을 보지 않아도 로디가 필요한 답장을 도와드립니다.

### 1차 경고

- 상태: `휴대폰 주의` / 위험도 `MEDIUM`
- 로디: "{{profileName}}, 방금 휴대폰 쪽으로 시선이 오래 갔어요. 전방을 봐주세요."
- 하단 패널: 없음

### 반복 감지 후 대행 처리

- 상태: `휴대폰 사용 반복`에서 `대행 처리 제안`으로 전환
- 로디: "지금 확인하려는 내용, 제가 대신 처리해드릴까요?"
- 선택지:
  - `대신 처리`
  - 사용자 대사: "석현이에게 20분정도 늦을 것 같다고 문자 보내줘."
- 하단 패널: 없음

### 문자 초안 확인

- 로디: "석현님에게 20분정도 늦을 것 같아 라고 보낼까요?"
- 선택지:
  - `보내기`
  - 사용자 대사: "응, 보내줘"
- 하단 패널:
  - 유형: 문자 초안
  - 제목: `석현님에게 보낼 메시지`
  - 메타: `문자 초안`
  - 내용: `20분정도 늦을 것 같아.`
  - 버튼: `보내기`

### 전송 완료

- 로디: "메시지를 보냈어요."
- 하단 패널:
  - 유형: 완료 액션
  - 제목: `메시지 전송 완료`
  - 메타: `문자 전송`
  - 내용: `석현님에게 메시지를 보냈습니다.`
  - 버튼: `확인`
- 비고: 화면상 완료만 표현하며 실제 문자 전송은 하지 않습니다.

### 종료

- 로디: "오늘 주행이 끝났어요. 운전 리포트를 정리해드릴게요."
- 결과: 운전 리포트 모달, 상세 리포트, 이메일 UI가 제공됩니다.

## 3. 기기조작 감지

설명: 주행 중 운전자가 음악 패널을 직접 조작하자 로디가 음악 변경을 대신 도와드립니다.

### 음악 패널 조작 및 1차 경고

- 상태: `기기 조작 주의` / 위험도 `MEDIUM`
- 로디: "{{profileName}}, 전방 확인이 계속 안 되고 있어요. 시선을 앞으로 돌려주세요."
- 하단 패널: 없음

### 반복 감지 후 조작 대행 제안

- 상태: `기기 조작 반복`에서 `조작 대행 제안`으로 전환
- 로디: "계속 화면을 조작하면 위험해요. 필요한 건 제가 대신 도와드릴까요?"
- 선택지:
  - `음악 바꿔줘.`
  - 사용자 대사: "지금 듣는 노래 말고 다른 음악으로 바꿔줘."
- 하단 패널: 없음

### 음악 설정

- 로디: "어떤 음악으로 바꿔드릴까요?"
- 선택지:
  - `빅뱅의 붉은 노을 틀어줘.`
  - 사용자 대사: "빅뱅의 붉은 노을 틀어줘."
- 하단 패널: 없음

### 음악 후보 확인

- 로디: "빅뱅의 붉은 노을을 찾았어요. 이 곡으로 재생할까요?"
- 선택지:
  - `응, 재생해줘.`
  - 사용자 대사: "응, 재생해줘."
- 하단 패널:
  - 유형: 음악 추천
  - 제목: `붉은 노을`
  - 메타: `빅뱅`
  - 내용: `요청한 곡을 찾아 재생할 준비를 마쳤어요.`
  - 버튼: `재생`

### 음악 재생 시작

- 로디: "빅뱅의 붉은 노을을 재생할게요."
- 하단 패널: 위와 같은 `붉은 노을` 음악 추천 패널이 유지됩니다.
- 미니 플레이어: `붉은 노을` / `빅뱅` 재생 상태를 표시합니다.

### 종료

- 로디: "오늘 주행이 끝났어요. 운전 리포트를 정리해드릴게요."
- 결과: 운전 리포트 모달, 상세 리포트, 이메일 UI가 제공됩니다.

## 4. 오늘은 크게 또박또박 말해줘

설명: 안내 음성을 크고 또박또박 바꾸는 미니 데모입니다. 공통 주행 준비를 생략합니다.

1. 로디: "좋아요. 현재 경로를 계속 안내할게요."
2. 사용자 대사: "로디야, 오늘 좀 피곤해서 크게 또박또박 말해줘"
3. 로디: "{{profileName}}, 알겠어요. 오늘은 평소보다 크게, 또박또박 들리게 안내할게요."
   - 효과: 이 단계부터 `FORMAL` 안내 음성 스타일을 적용합니다.
   - 하단 패널: 없음
4. 로디: "전방 주시를 유지해 주세요. 다음 안내는 필요한 순간에 분명하게 말씀드릴게요."
   - 효과: `FORMAL` 안내 음성 스타일 유지
   - 하단 패널: 없음
5. 로디: "오늘 주행이 끝났어요. 운전 리포트를 정리해드릴게요."
   - 결과: 개인화 데모는 리포트 모달, 상세 리포트, 이메일 UI를 열지 않습니다.

## 5. 시선 이탈 감지

설명: 시선이 벗어났을 때 전방 주시를 안내하는 미니 데모입니다. 공통 주행 준비를 생략합니다.

1. 상태: `시선 이탈 주의` / 위험도 `MEDIUM`
2. 로디: "전방을 봐주세요. 시선이 도로에서 오래 벗어났어요."
3. 사용자 대사: "알겠어, 앞 볼게."
4. 로디: "좋아요. 지금처럼 전방 주시를 유지해 주세요."
5. 로디: "오늘 주행이 끝났어요. 운전 리포트를 정리해드릴게요."
   - 결과: 미니 데모는 리포트 모달, 상세 리포트, 이메일 UI를 열지 않습니다.
6. 하단 패널: 전체 흐름에서 없음

## 6. 뒷좌석 확인 감지

설명: 뒤쪽 확인은 정차 후 하도록 안내하는 미니 데모입니다. 공통 주행 준비를 생략합니다.

1. 상태: `뒤돌아봄 주의` / 위험도 `MEDIUM`
2. 로디: "뒤쪽 확인할 일이 있으면 잠깐 멈춘 뒤에 보는 게 좋아요."
3. 사용자 대사: "응, 잠깐 정차하고 볼게."
4. 로디: "좋아요. 정차 후 확인하면 훨씬 안전해요."
5. 로디: "오늘 주행이 끝났어요. 운전 리포트를 정리해드릴게요."
   - 결과: 미니 데모는 리포트 모달, 상세 리포트, 이메일 UI를 열지 않습니다.
6. 하단 패널: 전체 흐름에서 없음

## 수정 시 참고할 현재 제약

- 선택지 라벨과 사용자 대사는 분리되어 있습니다. 라벨을 바꿀 때에는 연결된 사용자 대사도 함께 검토해야 합니다.
- 하단 패널은 이벤트 ID에 따라 별도로 연결됩니다. 대사만 바꿔도 패널 제목, 메타, 내용, 버튼은 자동으로 바뀌지 않습니다.
- 1~3번 대표 카드의 종료는 리포트 UI까지 이어지고, 4~6번 미니 카드는 리포트 UI 없이 종료됩니다.
